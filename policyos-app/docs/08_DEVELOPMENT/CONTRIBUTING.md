# Contributing

## Workflow
1. Create or select an approved issue/specification.
2. Create a focused feature branch.
3. Inspect current code and tests.
4. Implement a small coherent change.
5. Add tests.
6. Run Ruff and Pytest.
7. Update docs.
8. Commit with a clear message.
9. Open a pull request.
10. Resolve review findings.

## Commit examples
- `feat: add JWT access token utilities`
- `test: cover invalid password verification`
- `docs: define Sprint 2 authentication contract`
